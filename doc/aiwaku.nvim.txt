1. INTRODUCTION                                                   *aiwaku-intro*

aiwaku.nvim brings any CLI AI tool into Neovim as a persistent sidebar panel.

It does not implement AI features itself. Instead it acts as a bridge: you
configure the CLI command you already use (`copilot`, `claude`, `opencode`,
`aider`, or any other), and aiwaku runs it in a tmux session attached to a
Neovim terminal split.

Because sessions are backed by tmux, your conversation context persists across
panel toggles, window changes, and Neovim restarts. You can maintain multiple
independent sessions and switch between them with a picker.

==============================================================================
2. REQUIREMENTS                                             *aiwaku-requirements*

System dependencies:~

  tmux >= 3.0      Session persistence and management
                   https://github.com/tmux/tmux

  Neovim >= 0.10   Required for vim.system and modern nvim API

Neovim plugin dependencies:~

  nvim-lua/plenary.nvim    Async job execution
                           https://github.com/nvim-lua/plenary.nvim

  stevearc/dressing.nvim   (optional) Floating UI for vim.ui.select.
                           Without it Neovim falls back to the built-in
                           command-line prompt. Other providers (e.g.
                           telescope with the ui-select extension) also work.
                           https://github.com/stevearc/dressing.nvim

==============================================================================
3. INSTALLATION                                             *aiwaku-installation*

Using lazy.nvim: >lua

  {
    "juhaku/aiwaku.nvim",
    dependencies = {
        "nvim-lua/plenary.nvim",
        "stevearc/dressing.nvim", -- optional: floating UI for vim.ui.select
    },
    opts = { cmd = { "opencode" } },
  }
<

==============================================================================
4. SETUP                                                          *aiwaku-setup*

Call `require("aiwaku").setup(opts)` once during Neovim startup. The only
required option is `cmd`.

Minimal example: >lua

  require("aiwaku").setup({
    cmd = { "opencode" },
  })
<

`cmd` accepts a list (recommended) or a plain string: >lua

  cmd = { "/usr/local/bin/claude", "--model", "claude-3-5-sonnet" }
  cmd = "aider --model gpt-4o"
<

==============================================================================
5. CONFIGURATION                                           *aiwaku-configuration*

                                                            *aiwaku-config-cmd*
cmd ~
  Type: `string | string[] | CliTool[]`
  Default: `{ "copilot" }`

  The CLI tool(s) to run inside the tmux session. All formats are accepted:

  Single tool (old formats still work): >lua
    cmd = "aider --model gpt-4o"
    cmd = { "/usr/local/bin/claude", "--model", "claude-3-5-sonnet" }
<

  Multiple named tools: >lua
    cmd = {
      { name = "Claude", cmd = { "claude", "--dangerously-skip-permissions" } },
      { name = "Copilot", cmd = "copilot" },
    }
<
  The `name` field is optional; when absent it is resolved from the `cmd`
  basename. Use `select_tool()` to switch the active tool at runtime.

                                                          *aiwaku-config-width*
width ~
  Type: `integer`
  Default: `80`

  Width of the sidebar panel in columns.

                                                       *aiwaku-config-position*
position ~
  Type: `"right" | "left"`
  Default: `"right"`

  Which side of the screen to open the sidebar on.

                                                     *aiwaku-config-auto_submit*
auto_submit ~
  Type: `boolean`
  Default: `false`

  When `true`, automatically sends a carriage-return (Enter key) to the AI
  terminal after every `send_selection()` or `send_buffer()` call. This
  triggers the AI tool to process the input immediately, without the user
  pressing Enter manually. Useful for readline-based CLI tools such as
  `copilot-cli`, `claude`, and `opencode`.

  Enable in your setup call: >lua

    require("aiwaku").setup({
      auto_submit = true,
    })
<

                                                        *aiwaku-config-keymaps*
keymaps ~
  Type: `table`

  Map of mode lists to keymap definitions. Each entry maps a left-hand side
  string to a table with `command` (string or function) and `description`.

  Default: >lua

    keymaps = {
      [{ "n" }] = {
        ["<leader>ai"] = {
          command = function() require("aiwaku").toggle() end,
          description = "Toggle Aiwaku",
        },
        ["<leader>an"] = {
          command = function() require("aiwaku").new_session() end,
          description = "Aiwaku: new session",
        },
        ["<leader>as"] = {
          command = function() require("aiwaku").select_session() end,
          description = "Aiwaku: select session",
        },
        ["<leader>ar"] = {
          command = function() require("aiwaku").rename_session() end,
          description = "Aiwaku: rename session",
        },
        ["<leader>ab"] = {
          command = function() require("aiwaku").send_buffer() end,
          description = "Aiwaku: send buffer",
        },
        ["<leader>ad"] = {
          command = function() require("aiwaku").send_diagnostic() end,
          description = "Aiwaku: send diagnostic",
        },
        ["<leader>at"] = {
          command = function() require("aiwaku").select_tool() end,
          description = "Aiwaku: select CLI tool",
        },
      },
      [{ "v" }] = {
        ["<leader>ai"] = {
          command = "<Esc><Cmd>lua require('aiwaku').send_selection()<CR>",
          description = "Aiwaku: send selection",
        },
      },
    },
<

                                               *aiwaku-config-lsp_code_actions*
lsp_code_actions ~
  Type: `table[]`

  List of LSP code actions exposed through null-ls/none-ls. Each entry requires
  a `title` string and may include an optional `prompt` string. Entries with
  `buffer = true` dispatch to |aiwaku.send_buffer()| instead of
  |aiwaku.send_selection()|. Entries with `diagnostic = true` dispatch to
  |aiwaku.send_diagnostic()| and are only shown when the cursor line has a
  diagnostic. Entries with `file_diagnostic = true` dispatch to
  |aiwaku.send_file_diagnostics()| and are only shown when the buffer has at
  least one diagnostic.

  Default: >lua

    lsp_code_actions = {
      { title = "AI: send selection" },
      { title = "AI: explain this code", prompt = "explain this code:" },
      { title = "AI: refactor this code", prompt = "refactor this code:" },
      { title = "AI: send this file", buffer = true },
      { title = "AI: explain this file", prompt = "explain this file:", buffer = true },
      { title = "AI: send diagnostics", diagnostic = true },
      { title = "AI: fix diagnostics", prompt = "Fix the following diagnostics:\n", diagnostic = true },
      { title = "AI: send file diagnostics", file_diagnostic = true },
    },
<



                                               *aiwaku-config-terminal_keymaps*
terminal_keymaps ~
  Type: `table`

  Keymaps active only inside the sidebar terminal buffer. Keys map to a table
  with `command` (string) and `description`.

  Default: >lua

    terminal_keymaps = {
      ["<C-w>h"] = {
        command = "<C-\\><C-n><C-w>h",
        description = "Focus left",
      },
      ["<C-w>l"] = {
        command = "<C-\\><C-n><C-w>l",
        description = "Focus right",
      },
      ["<C-a>i"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').toggle()<CR>",
        description = "Toggle Aiwaku",
      },
      ["<C-a>r"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').rename_session()<CR>",
        description = "Aiwaku: rename session",
      },
      ["<C-a>s"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').select_session()<CR>",
        description = "Aiwaku: select session",
      },
      ["<C-a>n"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').new_session()<CR>",
        description = "Aiwaku: new session",
      },
      ["<C-a>c"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').clear_context()<CR>",
        description = "Aiwaku: clear context",
      },
      ["<C-a>t"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').select_tool()<CR>",
        description = "Aiwaku: select CLI tool",
      },
      ["<C-o>"] = {
        command = "<C-\\><C-n><Cmd>lua require('aiwaku').open_cword_in_tab()<CR>",
        description = "Aiwaku: open file under cursor in new tab",
      },
    },
<

==============================================================================
6. KEYMAPS                                                       *aiwaku-keymaps*

Default normal mode keymaps:~

  `<leader>ai`    Toggle the AI sidebar (open/close)
  `<leader>an`    Start a new session
  `<leader>as`    Select from existing sessions (picker)
  `<leader>ar`    Rename the current session
  `<leader>ab`    Send the current buffer to the AI
  `<leader>ad`    Send the diagnostic under cursor to the AI
  `<leader>at`    Select the active CLI tool (picker)

Default visual mode keymaps:~

  `<leader>ai`    Send the selected text to the AI

Default terminal mode keymaps (inside the sidebar):~

  `<C-w>h`        Move focus to the left window
  `<C-w>l`        Move focus to the right window
  `<C-a>i`        Toggle the AI sidebar (open/close)
  `<C-a>r`        Rename the current session
  `<C-a>s`        Select from existing sessions (picker)
  `<C-a>n`        Start a new session
  `<C-a>c`        Clear context (kill session and start fresh)
  `<C-a>t`        Select the active CLI tool (picker)
  `<C-o>`         Open file path under cursor in a new tab

All keymaps can be overridden or disabled by setting `keymaps` and
`terminal_keymaps` in your |aiwaku-setup| call. Pass an empty table to disable
all default keymaps.

7. API                                                             *aiwaku-api*

All functions are on the module returned by `require("aiwaku")`. They must be
called after |aiwaku-setup| has been invoked.

                                                             *aiwaku.setup()*
setup({opts})
  Initialize the plugin. Must be called once before any other function.
  {opts} is merged with the defaults; unset keys fall back to defaults.

                                                            *aiwaku.toggle()*
toggle()
  Open the sidebar if hidden, close it if visible.
  When opening, resumes the current session if one is active; creates a new
  session otherwise.

                                                       *aiwaku.new_session()*
new_session([{name}])
  Create a new AI session with an optional {name}. If {name} is omitted a
  unique auto-generated name is created using the active tool name, a random
  adjective-noun pair, and a hex suffix (e.g. "ai-claude-quirky-tesla-a7f3").
  Opens the sidebar showing the new session. Uses the tool selected via
  |aiwaku.select_tool()|; falls back to the first configured tool.

                                                    *aiwaku.select_session()*
select_session()
  Open a `vim.ui.select` picker listing all active aiwaku tmux sessions.
  Selecting an item opens that session in the sidebar.

                                                       *aiwaku.select_tool()*
select_tool()
  Open a `vim.ui.select` picker to change the active CLI tool. The selected
  tool is stored in state and used by all subsequent calls to
  |aiwaku.new_session()|. When only one tool is configured the picker is
  skipped and that tool is selected directly.

                                                    *aiwaku.rename_session()*
rename_session()
  Prompt (via `vim.ui.input`) to rename the current session. The "ai-" prefix
  is always preserved so sessions remain visible in the picker.

                                                    *aiwaku.clear_context()*
clear_context()
  Kill the current tmux session (terminating the AI process and its context)
  and immediately start a fresh session in its place.

                                                   *aiwaku.send_selection()*
send_selection([{prompt}])
  Send the current visual selection to the active AI session. Must be called
  from a visual-mode mapping so that selection marks are set.

  An optional {prompt} string is prepended before the selected text: >lua

    require("aiwaku").send_selection("Explain this code:")
    require("aiwaku").send_selection("Write tests for:")
<
  If the sidebar is not visible it is opened automatically before sending.

                                                   *aiwaku.send_buffer()*
send_buffer([{prompt}])
  Send all lines of the current buffer to the active AI session.
  A context header (filename and filetype) is automatically prepended.
  If the sidebar is not visible it is opened automatically.

                                                   *aiwaku.send_diagnostic()*
send_diagnostic([{prompt}])
  Send the LSP/diagnostic messages on the current cursor line to the active AI
  session. When no diagnostic exists on the cursor line, falls back to all
  diagnostics in the buffer. Issues a WARN notification when no diagnostics
  are found. Each diagnostic is formatted as:
  `[SEVERITY] [source] file:line: message`
  If the sidebar is not visible it is opened automatically.

                                                   *aiwaku.send_file_diagnostics()*
send_file_diagnostics([{prompt}])
  Send every diagnostic in the current buffer to the active AI session.
  Uses the same `[SEVERITY] [source] file:line: message` format as
  |aiwaku.send_diagnostic()|. Issues a WARN notification when no diagnostics
  are found. If the sidebar is not visible it is opened automatically.

                                                   *aiwaku.open_cword_in_tab()*
open_cword_in_tab()
  Open the file path under the cursor in a new tab. Strips optional :line
  or :line:col suffixes. Falls back to cwd-relative lookup if the path is
  not absolute. Issues a WARN notification if the file cannot be found.

                                                   *aiwaku.session_name()*
session_name()
  Return the display name of the active AI session with the "ai-" prefix
  stripped. Returns nil when no session is active or before setup() is
  called. Safe to call from statusline expressions at any time.

7.1 Session Naming                                      *aiwaku-session-naming*

Sessions are created with auto-generated human-readable names in the format: >

  ai-<tool>-<adjective>-<noun>-<hex>
<

Examples:
  • ai-claude-quirky-tesla-a7f3
  • ai-copilot-festive-hawking-c2e1
  • ai-opencode-mysterious-shannon-9f4b

The adjective-noun pair is drawn from a curated list of descriptive words and
scientists' names (Docker-style), providing ~4,880 base combinations. The
4-character hex suffix adds ~65k variants, yielding 319 million unique
combinations to prevent collisions across sessions.

You can pass a custom {name} to |aiwaku.new_session(name)| to override the
default naming.

8. LSP CODE ACTIONS                                        *aiwaku-code-actions*

aiwaku ships a null-ls source that exposes AI actions through the standard LSP
code action menu (`:lua vim.lsp.buf.code_action()`).

null-ls (or its community fork none-ls) must be installed and have an active
client attached to the buffer for actions to appear.

Setup:~

Register the source alongside your other null-ls sources: >lua

  local null_ls = require("null-ls")
  null_ls.setup({
    sources = {
      require("aiwaku.lsp-code-actions"),
    },
  })
<

Default actions:~

  AI: send selection              Send selection without a prompt prefix
  AI: explain this code       Prepend "explain this code:" before selection
  AI: refactor this code      Prepend "refactor this code:" before selection
  AI: send this file          Send full buffer without a prompt prefix
  AI: explain this file       Prepend "explain this file:" before buffer content
  AI: send diagnostics        Send cursor-line diagnostics (cursor must have a diagnostic)
  AI: fix diagnostics         Prepend fix prompt before cursor-line diagnostics (cursor must have a diagnostic)
  AI: send file diagnostics   Send all buffer diagnostics (shown when buffer has any diagnostic)

Actions without `buffer`, `diagnostic`, or `file_diagnostic` call
|aiwaku.send_selection()| internally. Actions with `buffer = true` call
|aiwaku.send_buffer()|. Actions with `diagnostic = true` call
|aiwaku.send_diagnostic()| and are only shown when the cursor line has a
diagnostic. Actions with `file_diagnostic = true` call
|aiwaku.send_file_diagnostics()| and are only shown when the buffer has at
least one diagnostic. The sidebar is opened automatically if not visible.

Overriding the action list:~

  You can replace the defaults during |aiwaku-setup|: >lua

    require("aiwaku").setup({
      cmd = { "opencode" },
      lsp_code_actions = {
        { title = "AI: send selection" },
        { title = "AI: write tests", prompt = "write tests for:" },
        { title = "AI: review this code", prompt = "review this code:" },
      },
    })
<

  Each entry requires a `title`. `prompt` is optional; when omitted, aiwaku
  sends the current selection without a prefix. Entries with `buffer = true`
  dispatch to |aiwaku.send_buffer()| instead of |aiwaku.send_selection()|.
  Entries with `diagnostic = true` dispatch to |aiwaku.send_diagnostic()|
  and are only shown when the cursor line has a diagnostic. Entries with
  `file_diagnostic = true` dispatch to |aiwaku.send_file_diagnostics()| and
  are only shown when the buffer has at least one diagnostic.

Actions are available for all filetypes when null-ls has an active client on
the buffer.

9. STATUSLINE INTEGRATION                                    *aiwaku-statusline*

|aiwaku.session_name()| returns the display name of the active session (with
the "ai-" prefix stripped), or nil when no session is active.

lualine example: >lua

  {
    function()
      return " " .. (require("aiwaku").session_name() or "")
    end,
    cond = function()
      return require("aiwaku").session_name() ~= nil
    end,
  }
<

Plain statusline example: >lua

  _G.AiwakuStatusline = function()
    local name = require("aiwaku").session_name()
    return name and (" " .. name) or ""
  end
  vim.o.statusline = "%{%v:lua.AiwakuStatusline()%} %f %=%l:%c"
<

The function is safe to call before setup() — it returns nil without error.
